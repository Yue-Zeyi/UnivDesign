## 视觉主题与氛围

这个界面说的是"笃定的从容"。每个表面之所以存在，是因为用户需要看到它——而非设计师想要装饰。信息密度本身就是美学：紧凑的行距、沉静的中性色、一抹蓝色承担所有交互承诺。设计借鉴了控制面板与仪表盘的语言——在那里，清晰是生存之本，而非偏好。

三根看不见的柱子撑起一切：

- **确定性** — 相同的操作永远产生相同的结果。主按钮永远是蓝色。危险状态永远是红色。用户永远不需要猜。
- **意义感** — 每一个像素都要证明自己存在的价值。如果去掉它毫无影响，它就不该出现。
- **自然性** — 交互流程模仿物理世界的逻辑。表单从上往下读。表格从左往右扫。确认永远跟在操作之后。

界面应该像一只整理得当的文件柜：不花哨，但一切都在你以为的位置。

## 色彩体系与角色

颜色不是装饰——它们是指令。每种颜色告诉用户该感受什么、该做什么。

| 角色 | 令牌 | 色值 | 为什么是这个颜色 |
|------|------|------|------------------|
| 主色 | colorPrimary | #1677FF | "点这里。" 一种自信而冷静的蓝色，表达了行动而非攻击。它是页面上唯一的饱和色，所以永远赢得注意力层级。 |
| 主色悬停 | colorPrimaryHover | #4096FF | "你快要点击了。" 更浅的蓝色确认了你的意图——按钮在回应你的靠近。 |
| 主色按下 | colorPrimaryActive | #0958D9 | "你刚按下了。" 更深的蓝色压缩了视觉，模拟物理按压。 |
| 成功 | colorSuccess | #52C41A | "一切正常。" 绿色是全球通行的"通行"信号。足够醒目但绝不尖叫。 |
| 警告 | colorWarning | #FAAD14 | "注意，但别慌。" 琥珀色占据安全与危险之间的中间地带。 |
| 错误 | colorError | #FF4D4F | "停下。出问题了。" 红色在任何文化中都意味着停止。仅用于问题和破坏性操作——如果红色出现，它必须要求立即关注。 |
| 主文本 | colorText | rgba(0,0,0,0.88) | 不是纯黑——88%的不透明度赋予文字重量却不带#000的刺眼感，在数据密集的视图中减轻眼部疲劳。 |
| 次文本 | colorTextSecondary | rgba(0,0,0,0.65) | "这是背景信息，不是主角。" 65%的不透明度创造层级而不引入新颜色。 |
| 辅助文本 | colorTextTertiary | rgba(0,0,0,0.45) | "需要时它在这里。" 占位符、时间戳、元数据——可见但永不争抢。 |
| 容器背景 | colorBgContainer | #FFFFFF | 白色是内容栖息的虚空。它说"这是一个你可以信任的表面"。 |
| 布局背景 | colorBgLayout | #F5F5F5 | "这是地板，不是家具。" 暖灰色将画布与内容分开而不引人注意。 |
| 边框 | colorBorder | #D9D9D9 | "这里有分隔。" 边框是最安静的分隔形式——因为它们不增加视觉重量。 |
| 次边框 | colorBorderSecondary | #F0F0F0 | "这些东西相关但不同。" 用于同一层级内的细微分隔。 |

### 深色模式

深色模式不是浅色模式的反色——它是不同的物理。同一高度上的表面变得更暗而非更亮。虚空变为#000000，容器漂浮在#141414，文字反转为rgba(255,255,255,0.85)。主色蓝保持不变，因为它在两种背景下都好用——选择它正是因为这个原因。

## 排版规则

字体是界面的骨架。排版失败，一切皆废。

字体：系统字体栈。不是因为自定义字体不好，而是因为系统字体渲染最快且在各平台都感觉原生。速度是一种设计决策。

代码：`'SFMono-Regular', Consolas, 'Liberation Mono', Menlo, Courier, monospace`

| 级别 | 字号 | 字重 | 行高 | 为什么 |
|------|------|------|------|--------|
| H1 | 38px | 600 | 1.3 | "这个页面是关于X的。" 展示标题是地标——大到足以定位，不至于大到喧宾夺主。 |
| H2 | 30px | 600 | 1.3 | "这个部分讲的是Y。" 分节标记创造节奏而不打断。 |
| H3 | 24px | 600 | 1.3 | "这个分组意味着Z。" 最小的标题，仍然感觉像标题。 |
| H4 | 20px | 600 | 1.3 | "这是重要上下文。" 分节中的强调标签。 |
| 正文 | 14px | 400 | 1.57 | 骨干。14px是密集企业界面中舒适阅读的下限。1.57行高留有呼吸空间而不浪费纵向空间。 |
| 小字 | 12px | 400 | 1.67 | "元数据住在这里。" 说明文字、时间戳、辅助文字。永远不要低于12px——更小的文字无法通过无障碍验证。 |

标题用600字重，因为700太吵。正文用400，因为它需要消失在阅读中。任何页面上只用两个字重——多了就是噪音。

## 组件样式

### 按钮（主色）
- 背景：主色蓝——因为这是你希望用户采取的操作
- 高度：32px默认，40px大，24px小——三种尺寸，一个节奏
- 内边距：4px 15px（默认）——紧凑因为企业屏幕是密集的
- 圆角：6px——圆到感觉友好，锐到感觉专业。不是药丸；不是矩形。
- 字体：14px，字重500——比正文略重以确认可点击性
- 悬停：主色悬停蓝——按钮说"我看到你了"
- 焦点：box-shadow 0 0 0 2px rgba(5,145,255,0.1)——柔和的蓝色光环，不是攻击性的圆环
- 禁用：不透明度0.5——可见但已虚化。按钮在那里，但通道已关闭。
- 按下：主色按下蓝——视觉压缩，模拟物理按压

### 卡片
- 背景：白色，漂浮在布局灰色之上
- 边框：无——高度来自阴影，而非线条。卡片会呼吸。
- 圆角：8px——曲线足够成为容器，不至于到好玩的感觉
- 内边距：24px——宽裕因为卡片承载真正的内容
- 阴影：分层，微妙——三层阴影创造物理抬升的印象而不引人注意阴影本身

### 输入框
- 高度：32px——匹配按钮高度以实现水平对齐
- 边框：1px实线边框灰——存在但不强势
- 圆角：6px——匹配按钮，因为输入框和按钮是兄弟
- 悬停：边框向主色变浅——"我准备好了"
- 焦点：主色边框+柔和光环——"我在听"
- 错误：错误红边框——毫不含糊，因为困惑的用户会放弃表单

### 表格
- 行高：55px——高到可扫视，紧凑到有密度
- 表头：#FAFAFA背景——"我是标签，不是数据。" 微妙的区分，不是激进的分隔。
- 悬停行：#FAFAFA——"你在看这一行。" 最安静的可能高亮。

## 布局原则

- 基础单位：4px。每个间距决策都是4的倍数。这不是随意的——4px创造一致的节奏而不需要微观管理。
- 间距比例：4、8、12、16、20、24、32、40、48、64。使用比例，不要用任意值。比例创造不同人构建的页面之间的视觉一致性。
- 最大内容宽度：1200px。超过这个，行太长，眼睛迷路。
- 网格：24列。足够细粒度适应任何布局，足够标准适应任何开发者。
- 圆角比例：2px（标签）→ 4px（小）→ 6px（按钮）→ 8px（卡片/弹窗）。每升一级意味着"更大的容器"。圆角告诉你正在看什么类型的元素。

## 深度与高度

阴影不是装饰——它们传递空间关系。有阴影的卡片在页面之上。有更多阴影的下拉菜单在卡片之上。

| 级别 | 用途 | 阴影 | 为什么 |
|------|------|------|--------|
| 0 | 页面背景 | 无 | 地面层。一切栖息于此。 |
| 1 | 卡片、容器 | 0 1px 2px rgba(0,0,0,0.03), 0 1px 6px -1px rgba(0,0,0,0.02) | 几乎看不到的抬升——卡片存在于空间中，但不宣告自己的高度。 |
| 2 | 下拉菜单 | 0 6px 16px rgba(0,0,0,0.08), 0 3px 6px -4px rgba(0,0,0,0.12) | 下拉菜单漂浮在内容之上。阴影更明显，因为元素是临时的，需要让人感觉脱离。 |
| 3 | 弹窗、对话框 | 0 6px 16px rgba(0,0,0,0.08), 0 9px 28px 8px rgba(0,0,0,0.05) | 最高高度对应最重要。弹窗占有用户的注意力。 |

## 宜与忌

宜：
- 每个区域的主色蓝只用于一件事：最重要的操作。如果一切都是蓝色，蓝色就失去了意义。
- 将字号保持在定义的比例内（12/14/16/20/24/30/38）。中间值会破坏节奏。
- 布局灰（#F5F5F5）用于页面背景，白色用于容器。这个双表面系统不用阴影就能创造深度。
- 为每个用户操作提供即时反馈。不响应的按钮感觉是坏的。
- 严格使用语义色——危险仅用于破坏性操作，警告仅用于注意，成功仅用于完成。

忌：
- 当令牌存在时不要硬编码hex值。令牌可以改变；硬编码值不行。
- 单个页面不要使用超过3个标题级别。更多创造噪音，而非层级。
- 不要在非破坏性操作中使用错误红。红色意味着停止——如果它也意味着"编辑"或"信息"，用户就不再信任它。
- 不要在24/32/40px比例之外覆盖组件高度。这三个尺寸让按钮、输入框和选择框对齐到同一个水平节奏。
- 不要让主色蓝与竞争性的强调色混用。一蓝。一强调。一层级。

## 响应式行为

| 断点 | 宽度 | 发生什么 |
|------|------|----------|
| xs | < 576px | 全宽，堆叠。界面变为单列滚动。 |
| sm | ≥ 576px | 2列网格成为可能。并列表单。 |
| md | ≥ 768px | 侧边栏折叠为覆盖层。界面在不丢失结构的情况下适配。 |
| lg | ≥ 992px | 标准仪表盘布局。侧边栏常驻。 |
| xl | ≥ 1200px | 完整布局。界面呼吸。 |

- 最小触摸目标：32px（默认组件高度）。在触控设备上使用大号（40px）变体。
- 字号永远不低于12px。低于12px，文字变成装饰，不再是沟通。

## 代理提示指南

快速调色板：primary=#1677FF, bg=#FFFFFF, layout-bg=#F5F5F5, text=rgba(0,0,0,0.88), border=#D9D9D9, success=#52C41A, warning=#FAAD14, error=#FF4D4F

### 即用提示词：
- "创建一个仪表盘页面" → 浅灰地板（#F5F5F5），白色卡片容器8px圆角，紧凑55px表格行，蓝色主按钮32px高度，14px正文，仅基于阴影的高度
- "构建一个带验证的表单" → 32px输入高度，6px圆角，蓝色焦点环，红色错误状态（#FF4D4F）配行内12px消息，绿色成功（#52C41A）用于确认
- "设计一个设置面板" → 白色卡片中的分组区块，使用主色蓝的切换开关，65%不透明度的次文本，24px卡片内边距，用边框而非阴影分隔
