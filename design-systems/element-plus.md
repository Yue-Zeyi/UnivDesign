## 视觉主题与氛围

这个界面说的是"沉静的自信"。干净的表面、极少的装饰、一抹蓝色承担所有交互承诺。没有装饰元素，因为装饰与内容竞争，而内容才是用户来到这里的理由。

四条原则，不多不少：

- **一致性** — 相同的模式，相同的结果。主按钮到处长得一样。表格行在每一页行为相同。用户不应该在第三个屏幕上学习新的视觉语言。
- **反馈** — 每个操作产生可见的响应。按钮确认悬停。表单在输入时验证。成功与错误状态即时且毫不含糊。沉默是唯一不可接受的反馈。
- **效率** — 减少步骤，而非减少功能。智能默认值、清晰的层级、预填表单。界面应该预判下一个操作，而非等待用户去找。
- **可控性** — 用户做选择。操作可逆。范围可见。界面永远不困住、不强迫、不假设。

界面应该像一张整理得当的书桌：不花哨，一切各就其位，无需思考即可触及。

## 色彩体系与角色

颜色是语义信号。每种颜色有一份工作描述，并且只做那份工作。

| 角色 | 令牌 | 色值 | 为什么是这个颜色 |
|------|------|------|------------------|
| 主色 | --el-color-primary | #409EFF | "点这里。" 一种天蓝色，感觉亲和但权威。它是页面上唯一的饱和色，确保它永远夺取焦点。 |
| 主色浅9 | --el-color-primary-light-9 | #ECF5FF | "这与主操作相关。" 一缕蓝色的低语——用于悬停状态和微妙高亮，近到表示关联，静到退居内容之后。 |
| 主色深2 | --el-color-primary-dark-2 | #337ECC | "你按下了。" 压缩的蓝色模拟物理按压。 |
| 成功 | --el-color-success | #67C23A | "完成。确认。已保存。" 绿色在任何语言中都意味着通行。够亮以注意，够柔不干扰。 |
| 警告 | --el-color-warning | #E6A23C | "继续前先看看这里。" 琥珀色位于安全与危险之间。它要求关注而非要求警觉。 |
| 危险 | --el-color-danger | #F56C6C | "停下。删除。撤销。" 红色专门留给破坏性或不可逆操作。如果红色出现，用户应该停顿。 |
| 信息 | --el-color-info | #909399 | "供你参考。" 中性的灰蓝色用于需要颜色但不配获得情绪的信息状态。 |
| 主文本 | --el-text-color-primary | #303133 | 接近黑色，但更暖。纯#000产生刺眼感；这种灰黑在密集布局中读起来更柔和。 |
| 常规文本 | --el-text-color-regular | #606266 | "这是支撑内容。" 中灰色用于描述和辅助信息——可见但从属。 |
| 次文本 | --el-text-color-secondary | #909399 | "这是上下文。" 时间戳、提示、元数据——需要时存在，永远不争抢。 |
| 占位文本 | --el-text-color-placeholder | #A8ABB2 | "这里会填东西。" 够浅以区别于已输入文本，够深以可读。 |
| 背景 | --el-bg-color | #FFFFFF | 值得信赖的表面。白色意味着"这是内容的容器"。 |
| 页面背景 | --el-bg-color-page | #F2F3F5 | "这是地板。" 暖灰色将画布与内容分开。它存在是为了让白色容器漂浮其上。 |
| 边框 | --el-border-color | #DCDFE6 | "东西在这里是分开的。" 边框是最安静的分隔形式——够显眼以构架，够微妙以忽略。 |
| 填充 | --el-fill-color-light | #F5F7FA | "这个区域略有不同。" 用于悬停状态、禁用背景和微妙区分。 |

### 深色模式

深色模式中，虚空是#141414。容器漂浮在#1D1D1D。文字反转为#E5EAF3。边框退至#4C4D4F。系统反转明度但保留关系——浅色模式中在上面的，深色模式中仍然在上面。

## 排版规则

字体创造界面的节奏。排版的一致性就是意义的一致性。

字体：系统字体栈配中文回退。速度与原生感优先于审美追求。增加200ms加载时间的自定义字体是设计失败。

| 级别 | 字号 | 字重 | 为什么 |
|------|------|------|--------|
| 超大 | 20px | 700 | 页面标题。每页只用一次。700字重锚定页面身份。 |
| 大 | 16px | 700 | 分节标题。700字重因为它们是结构地标，不是强调。 |
| 基准/正文 | 14px | 400 | 默认。14px是企业密度下舒适阅读的下限。400字重消失在阅读中。 |
| 小 | 13px | 400 | 紧凑标签。仍然可读，但承认空间有限。 |
| 超小 | 12px | 400 | 说明文字、脚注、时间戳。绝对下限——低于此文字不再是沟通。 |

只有两个字重重要：400用于正文，700用于强调。再多你就是在设计海报，不是界面。

## 组件样式

### 按钮（主色）
- 背景：主色蓝（#409EFF）——页面上最重要的操作
- 高度：32px默认，40px大，24px小——三种尺寸，一个纵向节奏
- 内边距：8px 16px——平衡标签可读性
- 圆角：4px——刻意的选择：圆到感觉可点击，方到感觉严肃。不好玩。不严厉。
- 字体：14px，字重500——比正文略重以确认交互性
- 悬停：#53A8FF——"我看到你靠近了"
- 按下：#337ECC——视觉压缩，按钮后退
- 禁用：#A0CFFF背景——虚化但可见，让用户知道选项存在即使不可用
- 朴素变体：浅9背景，浅5边框——用于需要主操作但不想主导的场合

### 卡片
- 背景：白色，漂浮在页面灰上
- 边框：1px实线#EBEEF5——一条细线，不是一堵墙。卡片被分隔，不是被囚禁。
- 圆角：4px——匹配按钮语言。圆角的一致性就是系统感的一致性。
- 内边距：20px——足够空间让内容呼吸
- 阴影：0px 0px 12px rgba(0,0,0,0.12)——需要时存在，不需要时隐形

### 输入框
- 高度：32px——匹配按钮实现水平对齐
- 边框：1px实线边框色——存在，不强势
- 圆角：4px——系统一致性
- 焦点：主色边框+内阴影——"我在听，请输入"
- 错误：危险色边框——毫不含糊，因为表单错误必须被纠正
- 禁用：#F5F7FA背景——字段在那里但门关了

### 表格
- 表头背景：#F5F7FA——"我是标签，不是数据。" 微妙区分。
- 悬停行：#ECF5FF（主色浅9）——一缕蓝色低语说"你在看这里"
- 边框：#EBEEF5——最薄的可能分隔

## 布局原则

- 网格：24列。足够细粒度适应任何布局。
- 间距模式：4、8、12、16、20、24、32、40。比例小因为企业界面是密集的。宽裕的间距是给营销页面的。
- 圆角比例：2px（标签）→ 4px（按钮/卡片）→ 20px（药丸）→ 100%（圆形）。每一步意味着不同的东西。
- 容器布局：头部60px，侧边栏300px，底部60px。这些是默认值，不是约束。

## 深度与高度

| 级别 | 用途 | 阴影 | 为什么 |
|------|------|------|--------|
| 基础 | 卡片、面板 | 0 2px 4px rgba(0,0,0,0.12), 0 0 6px rgba(0,0,0,0.04) | 微妙的抬升。卡片存在于空间中但不宣告自己。 |
| 浅 | 悬停状态 | 0 2px 12px rgba(0,0,0,0.12) | 元素在靠近时微微升起——"注意这里"的物理隐喻。 |
| 深 | 下拉菜单 | 0 2px 4px rgba(0,0,0,0.12), 0 0 6px rgba(0,0,0,0.12) | 临时元素需要更多阴影因为它们突然出现，必须感觉脱离。 |

## 宜与忌

宜：
- 使用CSS自定义属性（--el-*）进行主题化。它们级联、覆盖、在版本更新中存活。
- 使用主色浅9（#ECF5FF）作为悬停背景。够蓝以表示关联，够白以保持安静。
- 使用24/32/40px尺寸比例用于交互元素。这三个高度让按钮、输入框和选择框对齐到一个节奏。
- 尊重语义色。危险用于破坏。警告用于注意。成功用于完成。混淆这些信号会困惑用户。

忌：
- 不要用outline做焦点——本系统使用内阴影。outline制造视觉断裂。
- 令牌存在时不要硬编码hex值。令牌可以在一处覆盖；hex值必须逐个搜索。
- 不要使用超过一个强调色。系统围绕单一主色构建，由它生成所有变体。添加第二个强调色制造两个竞争层级。
- 不要在比例之外创建组件尺寸。28px的按钮破坏与32px输入框的对齐。
- 不要手动设置覆盖层的z-index。增量管理器从2000开始。手动值制造冲突。

## 响应式行为

| 断点 | 宽度 | 发生什么 |
|------|------|----------|
| xs | < 768px | 全宽，堆叠。侧边栏变为覆盖层。 |
| sm | ≥ 768px | 750px最大内容。2列布局。 |
| md | ≥ 992px | 970px最大。侧边栏可见。 |
| lg | ≥ 1200px | 1170px最大。完整桌面布局。 |
| xl | ≥ 1920px | 1170px最大配更宽边距。内容不会拉伸到无穷。 |

- 最小触摸目标：32px（默认尺寸）。大号变体（40px）用于触控优先界面。
- 字号永远不低于12px。
