## 视觉主题与氛围

这个界面说的是"温暖的专业"。设计语言根植于一个简单的理念：优秀的设计为每个人服务，在每台设备上，在每种场景中。每个表面都应该感觉它属于这里——手机上、平板上、桌面上、亮色中、暗色中、用鼠标、用手指。

三根柱子撑起系统：

- **光** — 既是物理的也是隐喻的。亚克力材质带来没有重量的深度。揭示高亮让表面在你靠近时感觉活着。界面在发光，不是只是待在那里。
- **深度** — 分层、透视和动效创造空间理解。元素不是扁平的——它们存在于空间中，在不同距离，有不同关系。
- **动效** — 有目的的动画引导注意力并传递因果。东西不是凭空出现；它们到达。它们不是凭空消失；它们离开。

视觉调性是"亲和的权威"。柔和圆角（6-8px），微妙的亚克力半透明，以及一种蓝色（#0078D4）说"你需要我时我在"而非"看我"。系统天然跨平台——它在每个操作系统上说同样的视觉语言。

### 深色模式

虚空是#1A1A1A。卡片漂浮在#2D2D2D。文字变为#F5F5F5。描边退至#5C5C5C。品牌蓝保持#0078D4因为它在两种背景上都经过测试——它在任何地方都好用。

## 排版规则

字体：Segoe UI Variable（Windows 11）。在其他平台上，系统回退到原生系统字体。优先保证平台原生感——用户应该感觉像在家，而不是在用一个移植应用。

| 级别 | 字号 | 字重 | 为什么 |
|------|------|------|--------|
| 展示 | 68px | 600 | "这是英雄声明。" 用于启动页和引导。巨大但受控。 |
| 标题大 | 28px | 600 | "这是页面标题。" 够大以定位，不至于大到压倒。 |
| 标题 | 20px | 600 | "这是分节标题。" 标准标题。 |
| 副标题 | 16px | 600 | "这是强调的标签。" 在标题和正文之间的边界上。 |
| 正文1 | 14px | 400 | 默认。14px在1.4行高下在密集布局中阅读舒适。 |
| 正文2 | 12px | 400 | 元数据的紧凑文字。 |
| 说明 | 12px | 400 | 可读尺寸的下限。 |

只有两个字重：400（常规）和600（半粗）。选择600而非700是因为半粗感觉自信而不具攻击性——它匹配系统"亲和的权威"调性。

## 组件样式

### 按钮（主色/品牌）
- 背景：#0078D4——"前行的路径"
- 高度：32px小，36px中，48px大
- 圆角：4px小，6px中，8px大——圆角随尺寸缩放
- 字体：14px，字重600——半粗匹配系统"自信但不攻击"的调性
- 悬停：#106EBE——"我看到你靠近了"
- 按下：#005A9E——视觉压缩
- 焦点："双环"——1px实线#000000内环，1px实线品牌色外环。双环是系统的无障碍签名。它始终可见因为键盘用户很重要。
- 禁用：不透明度0.38——虚化

### 卡片
- 背景：白色
- 圆角：8px——柔和但不是药丸形
- 阴影：随高度变化
- 亚克力：可选——用于需要半透明的卡片。亚克力用于临时表面（浮出面板、对话框），不是持久表面。

### 输入框
- 高度：36px中，48px大
- 圆角：6px（中），8px（大）
- 焦点：2px品牌蓝底边框——焦点指示器在底边，不是围绕整个控件。这是对标准焦点环的刻意偏离，因为底边焦点在视觉上干扰更少同时保持可访问。

## 布局原则

- 基础单位：4px
- 间距令牌：XXS (2px) → XS (4px) → SM (8px) → MD (12px) → LG (16px) → XL (20px) → XXL (28px)。命名而非编号，因为名称承载意义。
- 网格：12列
- 圆角比例：4px → 6px → 8px → 12px → 9999px（圆形）

## 深度与高度

本系统使用两种高度机制：阴影和亚克力材质。

| 级别 | 用途 | 处理 |
|------|------|------|
| 0 | 页面背景 | 无阴影，无亚克力 |
| 2 | 卡片、面板 | 微妙阴影+可选亚克力 |
| 4 | 浮出面板、下拉菜单 | 阴影+亚克力 |
| 8 | 对话框、弹窗 | 强阴影+亚克力 |

**亚克力材质**：背景模糊配噪点纹理覆盖。用于临时表面以维持空间上下文——用户可以看到浮出面板后面的内容，帮助他们理解自己在哪里。

**揭示高亮**：悬停时，交互元素显示跟随光标的微妙光边。这是系统标志性的"活着"效果——表面回应你的存在，不只是你的点击。

## 宜与忌

宜：
- 在临时表面使用亚克力材质。它维持空间上下文。
- 使用"双环"焦点指示器。内黑外品牌色。它在每个背景上都可见。
- 同时为鼠标和触控设计。48px大号用于触控，32px小号用于密集鼠标UI。
- 使用微妙动效引导注意力。东西应该到达，不是出现。

忌：
- 不要在持久表面使用亚克力。它只用于临时UI。持久亚克力制造性能问题和视觉噪音。
- 不要在非交互元素上应用揭示高亮。揭示意味着"你可以点这里"。
- 不要在矩形元素上使用全圆角。卡片最大8px。
- 不要在同一表面混用不同设计系统版本。
- 不要未经组织批准覆盖品牌蓝（#0078D4）。

## 响应式行为

| 断点 | 宽度 | 发生什么 |
|------|------|----------|
| 移动端 | < 640px | 单列，底部导航 |
| 平板 | 640-1024px | 轨道导航，2列 |
| 桌面 | ≥ 1024px | 完整布局，常驻导航 |
| 大屏 | ≥ 1920px | 扩展布局 |

- 最小触摸目标：44x44px（大按钮=48px）
- 字号永远不低于12px
