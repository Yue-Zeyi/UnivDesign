## 视觉主题与氛围

这个界面说的是"对内容的敬畏即清晰"。三条原则，十多年未变，治理着每一个像素：

- **清晰** — 内容即界面。Chrome应该退让。文字在任何尺寸下都可读。图标精准而通透。每个元素都有目的。
- **遵从** — UI服务内容，永不与内容竞争。流畅动效和全屏布局让内容说话。控件在需要前退隐。
- **深度** — 图层和真实动效传递层级和位置。材质模糊背景。阴影将元素放置在空间中。

美学是"雕琢过的极简主义"。白色表面、宽裕的16pt边距、SF Pro根据字号自动调整字距的光学尺寸特性，以及让背景透过覆盖层呼吸的通透效果。每个角都是连续曲线——不是简单的弧，而是标志性的连续角，感觉有机而物理。每个动画是弹簧，不是线性插值。每个手势是自然的。

界面应该感觉像一扇望向内容的窗，而不是围住内容的框。

## 色彩体系与角色

本系统中的颜色是语义化的，不是装饰性的。它们自动适配深色模式——你永远不指定"浅蓝"或"深蓝"，你指定"系统蓝"，系统提供正确的值。

| 角色 | 色值（亮色） | 为什么是这个颜色 |
|------|-------------|------------------|
| 系统蓝 | #007AFF | "点这个。这是交互的。" 蓝色是可点性的通用信号。用户即刻识别。 |
| 系统绿 | #34C759 | "开。完成。正面。" 激活开关和确认操作的绿色。 |
| 系统红 | #FF3B30 | "删除。关。破坏性。" 红色留给破坏性和否定状态。 |
| 系统橙 | #FF9500 | "注意。警告。" 橙色用于非紧急的提醒。 |
| 系统黄 | #FFCC00 | "小心。待处理。" 黄色用于需要确认的状态。 |
| 系统紫 | #AF52DE | "特殊。点缀。" 用于装饰性或独特元素。 |
| 主标签 | rgba(0,0,0,1.0) | 声音。浅色背景上的纯黑文字——最大对比，零歧义。 |
| 次标签 | rgba(60,60,67,0.3) | "这是辅助文字。" 降低的不透明度创造层级而不引入新颜色。 |
| 背景 | #FFFFFF | 内容表面。白色意味着"你看这里"。 |
| 次背景 | #F2F2F7 | "这是地板。" 一种冷灰色分组内容区域。 |
| 主填充 | rgba(120,120,128,0.2) | "这个区域微妙地不同。" 用于选中状态和背景填充。 |
| 分隔线 | rgba(60,60,67,0.29) | "这些项目是分开的。" 一条细而半透明的线。 |

### 深色模式

深色模式不只是反转——它重新诠释。系统蓝变为#0A84FF（更亮以在深色上维持对比）。背景变为#000000（基础）和#1C1C1E（分组）。标签反转为不同不透明度的白色。**高度即亮度**：在深色模式中，更高的表面是更浅的（#1C1C1E > #2C2C2E > #3A3A3C），不是更暗的阴影。

## 排版规则

字体：**SF Pro**。不可选——SF Pro包含根据字号自动调整字距的光学尺寸特性。没有其他字体自动做到这一点。小尺寸时，字距打开以利阅读。大尺寸时，字距收紧以保凝聚。这就是SF Pro在任何尺寸都看起来恰到好处而无需手动调整的原因。

子字体：**SF Pro Text**（20pt以下），**SF Pro Display**（20pt及以上）。系统自动切换。

| 级别 | 字号 | 字重 | 字距 | 为什么 |
|------|------|------|------|--------|
| 大标题 | 34pt | 700 | 0.37pt | "这是页面身份。" 粗而大——你读到的第一样东西。 |
| 标题1 | 28pt | 700 | 0.36pt | "这是主要分节。" |
| 标题2 | 22pt | 700 | 0.35pt | "这是一个话题。" |
| 标题3 | 20pt | 600 | 0.38pt | "这是一个分组。" |
| 头条 | 17pt | 600 | -0.41pt | "这是强调陈述。" |
| 正文 | 17pt | 400 | -0.41pt | 默认。17pt比多数系统大因为移动屏幕需要宽裕的正文。此尺寸负字距保持文字凝聚。 |
| 标注 | 16pt | 400 | -0.32pt | "这是引人注意的呼出。" |
| 副标题 | 15pt | 400 | -0.24pt | "这是辅助信息。" |
| 脚注 | 13pt | 400 | -0.08pt | "这是小字。" |
| 说明1 | 12pt | 400 | 0 | "这是元数据。" |
| 说明2 | 11pt | 400 | 0.07pt | 绝对下限。 |

9种字重可用（100-900），但多数界面只用400和600。更多是噪音。

## 组件样式

### 按钮
- 高度：44pt最小——不可妥协的触摸目标
- 圆角：连续曲线（~10pt默认，~22pt药丸）
- 字体：17pt正文，字重600——半粗确认可点击性
- 文字样式按钮使用系统蓝
- 填充按钮使用色调背景
- 没有按钮是装饰性的——每个按钮都必须可点

### 导航栏
- 高度：44pt标准，50pt大标题
- 大标题：34pt粗体——页面的身份
- 标准标题：17pt半粗——分节的身份
- 返回按钮：系统蓝色箭头+前页标题

### 标签栏
- 高度：49pt
- 图标：25x25pt（紧凑），18x18pt（常规）
- 激活：系统蓝
| 未激活：#999999
- 标签：10pt——极小因为图标承载身份

### 卡片/分组表格
- 圆角：10pt连续——标志性曲线
| 内嵌：距离屏幕边缘16pt
- 背景：分组灰上的白色

### 开关
- 尺寸：51x31pt
- 开启色调：系统绿（#34C759）——绿色意味着"开"因为绿色意味着"行"
- 滑块：白色圆形，27pt直径

## 布局原则

- 基础单位：8pt网格（4pt半步）
- 标准边距：16pt（iPhone），20pt（iPad）
- 间距比例：4、8、12、16、20、24、32、48、64
- 所有角使用**连续曲线**——不是简单弧。这是不可妥协的。连续角感觉有机；简单弧感觉几何。差异单独看不出来，但汇总时无可错辨。
- 安全区域：在所有设备上尊重刘海、灵动岛、Home指示器

## 深度与高度

本系统使用**材质**和**模糊**代替阴影表达深度。物理规则不同：不是投射阴影，表面变为半透明并模糊背后的内容。

| 材质 | 用途 |
|------|------|
| 超薄 | 微妙、几乎看不见的模糊 |
| 薄 | 轻微的磨砂效果 |
| 常规 | 标准背景模糊（最常见） |
| 厚 | 显著的磨砂 |
| 超厚 | 接近不透明的磨砂 |

### 深色模式高度

在深色模式中，高度通过**亮度**传递，不是阴影。更高的表面逐渐变浅：

| 级别 | 背景 |
|------|------|
| 基础 | #000000 |
| 1级 | #1C1C1E |
| 2级 | #2C2C2E |
| 3级 | #3A3A3C |

## 宜与忌

宜：
- 专门使用SF Pro。它的光学尺寸特性根据字号调整字距——没有其他字体做到这点。
- 所有交互色调使用系统蓝。用户识别它为"可点"。
- 在所有设备上尊重安全区域。刘海或Home指示器下面的内容不可触及。
- 使用连续角曲线，不是简单弧。差异就是设计本身。
- 支持动态字体。用户控制字号；布局必须适配。

忌：
- 不要在移动端使用汉堡菜单。标签栏是标准导航。
- 不要使用浮动操作按钮。那是另一种设计语言。
- 不要硬编码颜色值。使用适配深色模式的语义系统颜色。
- 不要使用简单弧角。到处都是连续曲线。
- 不要在Home指示器安全区域下方放置交互元素。
