## 视觉主题与氛围

这个界面说的是"克制的丰盈"。足够的视觉多样性应对复杂企业场景，但绝不为了装饰而装饰。深色模式是一等公民，不是事后的补丁——因为构建工具的人常常在夜间工作。

设计偏爱精确：每个令牌都是10步色阶，每种颜色都从种子算法推导而来。这不是强迫症——这是保险。当设计师需要一个更浅或更深的变体时，它已经存在了。没有猜测，没有"差不多"，没有凭肉眼选的hex值。

当一些系统偏向机构化时，这个偏向现代。数据视图中更紧凑的间距，更大胆的色彩对比，更刻意的留白。圆角略微更柔和。灰色略微更冷。阴影略微更分明。这些都是微观决策，但它们汇聚成一个感觉精确而不冰冷的系统。

## 色彩体系与角色

每种颜色都作为10步色阶存在。色阶是系统最重要的设计决策——它意味着永远有一个更浅或更深的变体可用，无需插值。

| 角色 | 令牌 | 色值 | 为什么是这个颜色 |
|------|------|------|------------------|
| 主色 | --color-primary-6 | #165DFF | "在这里行动。" 一种鲜明自信的蓝色，位于10步色阶的第6步——色阶的重心。够亮以在白色上跳出，够饱和以在灰色上站稳。 |
| 主色悬停 | --color-primary-5 | #4080FF | "你正在靠近操作。" 更浅的蓝色确认用户意图。 |
| 主色按下 | --color-primary-7 | #0E42D2 | "你按下了。" 更深的蓝色压缩视觉。 |
| 主色最浅 | --color-primary-1 | #E8F3FF | "这与主操作相关。" 淡到几乎读不出蓝色的色彩——完美用于悬停背景和选中状态。 |
| 主色最深 | --color-primary-10 | #041C4C | "这是最深的权威。" 用于浅色背景上的高对比文本。 |
| 成功 | --color-success-6 | #00B42A | "确认。完成。通行。" 一种明亮、毫不含糊的绿色。 |
| 警告 | --color-warning-6 | #FF7D00 | "前方注意。" 橙色而非黄色，因为黄色在白色上通不过对比度检测。 |
| 危险 | --color-danger-6 | #F53F3F | "停下。删除。摧毁。" 红色留给不可逆操作。 |
| 主文本 | --color-text-1 | #1D2129 | 接近黑色，但微蓝。冷调黑色在数据密集界面中比暖调黑色更显精致。 |
| 常规文本 | --color-text-2 | #4E5969 | "这是支撑内容。" 冷灰色用于描述。 |
| 辅助文本 | --color-text-3 | #86909C | "这是上下文。" 元数据、时间戳、提示。 |
| 禁用文本 | --color-text-4 | #C9CDD4 | "这个选项存在但不可用。" 够亮知道它在，够暗知道它不能用。 |
| 背景 | --color-bg-1 | #FFFFFF | 内容表面。白色意味着"信任这个容器"。 |
| 次背景 | --color-bg-2 | #F7F8FA | "这是地板。" 冷灰色——比竞争对手的暖灰略蓝，匹配系统的温度。 |
| 边框 | --color-border | #E5E6EB | "分隔，不是割裂。" 边框构建结构而不围困。 |

### 深色模式

深色模式不是补丁——它是平行宇宙。虚空是#17171A，容器漂浮在#232326，文字变为#F5F5F5。10步色阶自然反转，所以主色蓝仍然是主色蓝。这就是算法色阶的意义：它们适配，不会断裂。

## 排版规则

字体：系统字体栈配中文回退。优先保证CJK字符的渲染品质，而非审美个性。

| 级别 | 字号 | 字重 | 为什么 |
|------|------|------|--------|
| 标题1 | 20px | 500 | 页面标题。500字重（非700）因为这个系统偏爱安静的权威而非呐喊。 |
| 标题2 | 16px | 500 | 分节标题。够重以划分，够轻以不主导。 |
| 标题3 | 14px | 500 | 分组标签。最小的"标题"——其实是一个强调标签。 |
| 正文1 | 14px | 400 | 默认文字。14px是企业密度的标准。 |
| 正文2 | 13px | 400 | 密集数据视图的紧凑文字。1px的缩减在大规模中节省空间。 |
| 正文3 | 12px | 400 | 元数据、时间戳、说明文字。绝对下限。 |

字阶刻意精简——6个级别，不是9个，不是12个。每个级别必须证明自己存在的理由。如果两个级别服务同一目的，其中一个就得死。

## 组件样式

### 按钮（主色）
- 背景：主色蓝（#165DFF）
- 高度：32px默认，24px小，40px大
- 圆角：4px默认，8px大——更大的容器获得更大的圆角。圆角与元素成比例。
- 字体：14px，字重500——强调但非粗体
- 悬停：#4080FF——"我看到你了"
- 焦点：0 0 0 2px rgba(22,93,255,0.2)——柔和的光环，不是攻击性的圆环
- 按下：#0E42D2——视觉按压
- 禁用：不透明度0.4——虚化。选项存在但不可达。

### 卡片
- 背景：白色
- 边框：1px实线边框灰——用线条分隔，而非阴影。这个系统偏好基于边框的深度。
- 圆角：8px——比按钮更柔和，因为卡片更大，需要成比例更多的弧度
- 阴影：默认无。深度来自边框。阴影留给浮动元素。

### 输入框
- 高度：32px默认
- 圆角：4px——匹配按钮
- 焦点：主色边框+0 0 0 2px rgba(22,93,255,0.2)——与按钮焦点语言一致
- 错误：危险红边框——毫不含糊

### 表格
- 表头：--color-fill-1——"我是标签。" 最微妙的区分。
- 悬停行：--color-fill-2——"你在看这里。" 几乎看不见的高亮。
- 边框：--color-border-light——最薄的分隔

## 布局原则

- 基础单位：4px。每个测量值都是4的倍数。
- 间距比例：4、8、12、16、20、24、32、40、48、64——可用令牌--spacing-1至--spacing-22。
- 网格：24列。
- 圆角比例：2px（小）→ 4px（默认）→ 8px（卡片）→ 12px（弹窗）→ 50%（圆形）。圆角标识元素类型。

## 深度与高度

本系统偏好用边框做分隔，用阴影做浮动。区别很重要：边框说"这些是相邻的"，阴影说"这在上面"。

| 级别 | 用途 | 阴影 | 为什么 |
|------|------|------|--------|
| 1 | 卡片 | 0 2px 12px rgba(0,0,0,0.06) | 最轻的可能抬升。卡片在页面之上但不在用户之上。 |
| 2 | 下拉菜单 | 0 4px 20px rgba(0,0,0,0.08) | 临时元素需要更多存在感，因为它们突然出现。 |
| 3 | 弹窗 | 0 8px 40px rgba(0,0,0,0.12) | 弹窗占有整个房间。最高高度对应最大权威。 |
| 4 | 高浮出层 | 0 12px 48px rgba(0,0,0,0.16) | 罕见。用于必须抢占注意力的浮出面板。 |

## 宜与忌

宜：
- 每种语义色都使用10步色阶。算法推导保证一致性——不用肉眼选hex值。
- 相邻表面优先使用边框分隔而非阴影深度。
- 深色模式设计为一等变体。10步色阶自然处理反转。
- 数据密集视图中使用更紧凑的间距。企业屏幕牺牲舒适换取密度。

忌：
- 不要在10步色阶之外使用颜色。如果你需要两步之间的颜色，说明色阶错了——修正色阶，不要插值。
- 不要在矩形数据容器上使用大圆角。数据要直边；交互元素要弧线。
- 不要在同一表面层级中混用暖灰和冷灰。本系统使用冷灰。暖灰制造不协调。
- 不要在交互元素上使用渐变。渐变暗示按钮不该有的维度感。
- 单页不要超过3种字重。更多字重制造视觉噪音，而非强调。

## 响应式行为

| 断点 | 宽度 | 发生什么 |
|------|------|----------|
| xs | < 576px | 单列，堆叠 |
| sm | ≥ 576px | 2列网格 |
| md | ≥ 768px | 侧边栏覆盖层 |
| lg | ≥ 992px | 标准布局 |
| xl | ≥ 1200px | 完整布局，常驻导航 |

- 最小触摸目标：32px。触控设备上使用40px变体。
- 字号永远不低于12px。
