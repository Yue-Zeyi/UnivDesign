## 视觉主题与氛围

这个界面说的是"包容的专业"。它被设计为不可能出错。系统覆盖每一个主要平台——Web、移动端、小程序、桌面——每一像素必须在所有平台上保持一致。这不是野心；这是必要。当你的设计系统服务十亿用户、横跨十余平台，一致性不是特性——它是契约。

视觉调性是"亲和的权威"。表面是温暖明亮的。蓝色主色（#0052D9）感觉稳定而非活跃——它说"信任我"而非"点我"。圆角（8px默认）柔化了企业的锋芒而不让它感觉好玩。8px的卡片圆角是一个刻意的信号：锐角说"企业"，圆角说"平台"，这个系统选择了"平台"。

## 色彩体系与角色

| 角色 | 令牌 | 色值 | 为什么是这个颜色 |
|------|------|------|------------------|
| 主色 | --td-brand-color | #0052D9 | "这是前行的路径。" 一种深沉、值得信赖的蓝色。不是天蓝（太随意），不是海军蓝（太威权）。这种蓝色说"可靠"。 |
| 主色悬停 | --td-brand-color-hover | #0957D9 | "你正在靠近路径。" 几乎看不出变浅的蓝色确认意图。 |
| 主色按下 | --td-brand-color-active | #003CAB | "你踏上了路径。" 更深的蓝色模拟压缩。 |
| 主色浅 | --td-brand-color-light | #ECF2FE | "这与主操作相关。" 蓝色的低语用于背景和悬停状态。 |
| 成功 | --td-success-color | #2BA471 | "完成。确认。已保存。" 一种平衡的绿色——不霓虹，不暗淡。 |
| 警告 | --td-warning-color | #E37318 | "请留意前行。" 焦橙色而非警示黄。橙色在白色背景上比黄色更抓注意力。 |
| 危险 | --td-error-color | #D54941 | "停下。删除。撤销。" 一种温暖的红色——不是消防车红，那样会让人恐慌。这种红色说"小心"，不是"惊慌"。 |
| 主文本 | --td-text-color-primary | rgba(0,0,0,0.9) | 90%不透明度，不是纯黑。10%的透明度创造更柔和的文字，在长时间使用中减轻眼部疲劳。 |
| 次文本 | --td-text-color-secondary | rgba(0,0,0,0.6) | "这是辅助信息。" 60%不透明度创造清晰层级而不引入新颜色。 |
| 占位文本 | --td-text-color-placeholder | rgba(0,0,0,0.35) | "这里会放东西。" 够浅以区别于已输入文本，够深以可读。 |
| 容器背景 | --td-bg-color-container | #FFFFFF | 内容表面。白色意味着"这是一个你可以信任的容器"。 |
| 页面背景 | --td-bg-color-page | #F3F3F3 | "这是地板。" 中性灰色让容器漂浮。 |
| 边框 | --td-component-border | #DCDCDC | "分隔。" 中灰边框构建结构而不围困。 |

### 深色模式

虚空是#1B1B1B。容器漂浮在#242424。文字反转为rgba(255,255,255,0.9)。品牌蓝变浅为#4582E6，因为白色上好用的蓝在深色上不好用——系统是适配，不是简单反转。

## 排版规则

字体：系统字体栈配中文回退。跨平台一致性比审美个性更重要。

| 级别 | 字号 | 字重 | 为什么 |
|------|------|------|--------|
| 展示 | 36px | 600 | "这个页面存在。" 每个视图使用一次，用于顶层身份。 |
| 标题L | 28px | 600 | "这是一个主要分节。" 够大以划分，够精致以不主导。 |
| 标题M | 24px | 600 | "这是一个话题。" 标准分节标题。 |
| 标题S | 20px | 600 | "这是一个分组。" 密集布局的紧凑标题。 |
| 正文L | 16px | 400 | "这需要强调。" 用于不是标题的重要正文。 |
| 正文M | 14px | 400 | 默认。14px是企业标准。 |
| 正文S | 12px | 400 | 元数据。可读尺寸的下限。 |

所有标题使用600字重——不是700，因为这个系统看重平静胜过冲击。标题应该定位，不是呐喊。

## 组件样式

### 按钮（主色）
- 背景：#0052D9——"稳定的路径"
- 高度：32px默认，24px小，40px大
- 圆角：6px默认，3px小，9px大——圆角随尺寸缩放。更大的元素需要成比例更多弧度才感觉平衡。
- 字体：14px，字重500
- 悬停：#0957D9——"我确认你的靠近"
- 焦点：0 0 0 3px rgba(0,82,217,0.2)——比多数系统更宽的焦点环（3px），因为本系统优先考虑所有平台的无障碍
- 禁用：品牌色30%不透明度——虚化而非抹去

### 卡片
- 背景：白色
- 边框：1px实线#E7E7E7——边框构建结构。本系统同时使用边框和阴影，因为跨平台一致性需要不依赖阴影渲染质量的可靠视觉分隔。
- 圆角：8px——"平台"圆角。比企业标准的4px更柔和，因为亲和力很重要。
- 阴影：0 1px 10px rgba(0,0,0,0.05)——几乎不存在。卡片是被抬升的，不是漂浮的。

### 输入框
- 高度：32px默认
- 圆角：6px——匹配按钮
- 焦点：主色边框+0 0 0 2px rgba(0,82,217,0.2)——"我在听"
- 错误：危险色边框——毫不含糊

## 布局原则

- 基础单位：4px。--td-spacer令牌（1-8）覆盖4-32px。
- 间距比例：4、8、12、16、20、24、28、32
- 网格：12列
- 圆角比例：3px（标签）→ 6px（按钮）→ 9px（卡片）→ 12px（弹窗）→ 999px（药丸）。非标准的9px卡片圆角是有意的——在更大容器上比例更柔和。

## 深度与高度

| 级别 | 用途 | 阴影 | 为什么 |
|------|------|------|--------|
| 1 | 卡片、面板 | 0 1px 10px rgba(0,0,0,0.05) | 最微妙的抬升。卡片在地板之上但不宣告高度。 |
| 2 | 下拉菜单 | 0 3px 14px 2px rgba(0,0,0,0.05) | 临时元素需要更多阴影以感觉脱离。 |
| 3 | 弹窗 | 0 8px 20px rgba(0,0,0,0.06) | 弹窗占有整个房间。 |
| 4 | 浮层面板 | 0 12px 32px 4px rgba(0,0,0,0.08) | 最高高度对应最重要。 |

## 宜与忌

宜：
- 使用基于rgba的文字颜色。它们在不同背景上自然适配——无需定义单独的深色模式文字颜色。
- 卡片保持8px圆角。更柔和的角是品牌信号。
- 同时支持浅色和深色模式。系统服务于所有场景。
- 使用蓝色（#0052D9）表达稳定。它是"信任"蓝，不是"行动"蓝。

忌：
- 不要引入竞争性强调色。每产品一个品牌色。
- 不要在交互元素上使用锐角（0-2px）。6px最低限度维持"亲和"身份。
- 不要在24/32/40px比例之外覆盖组件高度。这三个尺寸跨平台创造一致节奏。
- 不要在同一表面层级中混用暖灰和冷灰底色。选一种温度并坚持。
- 当--td-*令牌存在时不要硬编码hex值。

## 响应式行为

| 断点 | 宽度 | 发生什么 |
|------|------|----------|
| xs | < 576px | 单列，堆叠 |
| sm | ≥ 576px | 2列布局 |
| md | ≥ 768px | 侧边栏覆盖层 |
| lg | ≥ 992px | 完整布局配侧边栏 |
| xl | ≥ 1200px | 宽布局 |

- 最小触摸目标：32px（移动端44px）
- 字号永远不低于12px

## 代理提示指南

快速调色板：primary=#0052D9, bg=#FFFFFF, page=#F3F3F3, text=rgba(0,0,0,0.9), border=#DCDCDC, success=#2BA471, warning=#E37318, danger=#D54941

### 即用提示词：
- "创建一个管理后台" → #F3F3F3页面背景，白色卡片8px圆角，蓝色#0052D9用于主操作，悬停行#ECF2FE，14px正文，6px按钮圆角
- "构建一个多平台表单" → 32px输入框，6px圆角，蓝色焦点环（0 0 0 3px rgba(0,82,217,0.2)），危险#D54941用于错误
- "设计一个数据概览" → 紧凑表格配#F3F3F3表头，白色容器，微妙阴影（0 1px 10px rgba(0,0,0,0.05)），品牌浅色#ECF2FE用于徽章
