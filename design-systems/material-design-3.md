## 视觉主题与氛围

这个界面说的是"表达性的个性化"。哲学是从僵硬的统一到温暖、有机、深度个人化设计的转向。系统不告诉你界面应该长什么样——它给你一颗种子，界面从种子生长出来。

三个原则，都指向内在：

- **表达性** — 色彩和形态适配你的内容和品牌，而不是反过来。你的UI应该像你，不像模板。
- **个人化** — 从种子生成动态色彩意味着每个用户的界面是独一无二属于他们的。没有两个体验需要看起来一样。
- **自适应性** — 组件在手机、平板、桌面和折叠设备上重塑自身而不丢失身份。设计是流动的，不是固定的。

美学借鉴杂志布局：大号展示字体、药丸形按钮、充裕的圆角、以及用柔和表面色调代替阴影来表达深度。系统说："你的界面应该感觉像一本个性化杂志，而不是一份表格。"

## 色彩体系与角色

本系统中的颜色不是静态的——它从种子生成。以下数值是基线（默认紫色种子），但每个值都可以根据用户的色彩来源改变。

### 亮色主题（基线）

| 角色 | 令牌 | 色值 | 为什么是这个颜色 |
|------|------|------|------------------|
| 主色 | --md-sys-color-primary | #6750A4 | "这是你的强调色。" 一种承载个性的丰富紫色。不是蓝色——蓝色属于企业。紫色属于自我表达。 |
| 主色上 | --md-sys-color-on-primary | #FFFFFF | 主色上的白色文字。对比度由算法保证。 |
| 主色容器 | --md-sys-color-primary-container | #EADDFF | "这是主色调色的表面。" 用于色调按钮和微妙填充。比主色浅但不可否认地相关。 |
| 主色容器上 | --md-sys-color-on-primary-container | #21005D | "这段文字在主色容器上。" 浅表面上的深文字。 |
| 辅色 | --md-sys-color-secondary | #625B71 | "这是支撑强调色。" 不如主色突出，用于色调区分。 |
| 辅色容器 | --md-sys-color-secondary-container | #E8DEF8 | "辅色调色的表面。" 用于中等强调元素。 |
| 三色 | --md-sys-color-tertiary | #7D5260 | "这是装饰性强调色。" 用于需要颜色但不需要层级的元素。 |
| 错误 | --md-sys-color-error | #B3261E | "出了问题。" 红色，普遍且毫不含糊。 |
| 错误容器 | --md-sys-color-error-container | #F9DEDC | "柔和的错误表面。" 用于不大叫的错误背景。 |
| 表面 | --md-sys-color-surface | #FFFBFE | "这是组件表面。" 几乎白色——主色的最淡色调。 |
| 表面上 | --md-sys-color-on-surface | #1C1B1F | "这段文字在表面上。" 近黑色，最大可读性。 |
| 表面变体 | --md-sys-color-surface-variant | #E7E0EC | "这个表面略有不同。" 用于需要从基础表面突出的卡片和容器。 |
| 轮廓 | --md-sys-color-outline | #79747E | "这是边框。" 一种中调色分隔而不割裂。 |

### 深色主题

算法反转色调色板。主色变为#D0BCFF。表面变为#1C1B1F。关系保持——浅色模式中在上面的，深色模式中仍然在上面。**色调高度**取代阴影：更高的表面获得渐亮的主色调，而非更深的阴影。

## 排版规则

字体：Roboto（默认）。系统定义比例，而非字体——任何字体都可以遵循这个比例。

| 级别 | 字号 | 字重 | 行高 | 字间距 | 为什么 |
|------|------|------|------|--------|--------|
| 展示大 | 57px | 400 | 64px | -0.25px | "这是英雄时刻。" 用于启动页和营销。紧凑字距（-0.25px）让大字保持凝聚。 |
| 展示中 | 45px | 400 | 52px | 0 | "这是一个重要声明。" |
| 展示小 | 36px | 400 | 44px | 0 | "这是一个突出的标题。" |
| 标题大 | 32px | 400 | 40px | 0 | "这是分节英雄。" |
| 标题中 | 28px | 400 | 36px | 0 | "这是分节标题。" |
| 标题小 | 24px | 400 | 32px | 0 | "这是子分节。" |
| 题目大 | 22px | 400 | 28px | 0 | "这是卡片标题。" |
| 题目中 | 16px | 500 | 24px | 0.15px | "这是强调标签。" 中等字重+微字距=自信但不吵。 |
| 正文大 | 16px | 400 | 24px | 0.5px | "这是舒适的阅读文字。" |
| 正文中 | 14px | 400 | 20px | 0.25px | 默认。14px配宽松字距以利阅读。 |
| 正文小 | 12px | 400 | 16px | 0.4px | "这是紧凑文字。" 元数据、说明。 |
| 标签大 | 14px | 500 | 20px | 0.1px | "这是按钮标签。" 中等字重确认可点击性。 |
| 标签小 | 11px | 500 | 16px | 0.5px | "这是微型标签。" 用于芯片和徽章。绝对下限。 |

15级字阶是雄心勃勃的——每个屏幕用3-4级。更多制造噪音。

## 组件样式

### 按钮——填充（主色）
- 背景：主色——"这是你想要的操作"
- 高度：40dp
- 圆角：20dp——**药丸形**。这是最重要的视觉签名：按钮不是圆角矩形，它们是药丸。这信号着温暖和亲和。
- 字体：标签大（14px，字重500）
- 状态层：悬停8%覆盖，焦点12%，按下12%——微妙的透明度变化代替颜色变化

### 按钮——色调
- 背景：辅色容器——"这是次要操作"
- 文字：辅色容器上
- 同样药丸形，同样尺寸

### 卡片——提升
- 背景：表面
- 圆角：12dp——充裕弧度营造有机感
- 阴影：1级——微妙抬升

### 卡片——填充
- 背景：表面变体——色调区分，无需阴影
- 圆角：12dp

### 卡片——轮廓
- 背景：表面
- 边框：1px轮廓——当你需要边界时
- 圆角：12dp

### 浮动操作按钮（FAB）
- 背景：主色容器
- 尺寸：56dp标准，40dp小，96dp大
- 圆角：16dp（标准）——不是圆形，略微圆角
- 高度：3级——它漂浮

## 布局原则

- 基础单位：4dp网格
- 间距：4、8、12、16、24、32、48
- 网格：4列（紧凑）→ 8列（中等）→ 12列（展开）
- 圆角比例：0dp → 4dp → 8dp → 12dp → 16dp → 20dp → 28dp → 50%。宽范围支持系统的有机美学。

### 断点

| 断点 | 宽度 | 列数 | 导航 |
|------|------|------|------|
| 紧凑 | < 600dp | 4 | 底部导航栏 |
| 中等 | 600-839dp | 8 | 导航轨道（侧面） |
| 展开 | ≥ 840dp | 12 | 导航抽屉 |

## 深度与高度

| 级别 | 阴影 | 为什么 |
|------|------|--------|
| 0 | 无 | 地面。 |
| 1 | 0 1px 2px rgba(0,0,0,0.3), 0 1px 3px 1px rgba(0,0,0,0.15) | 静止的提升卡片。 |
| 2 | 0 1px 2px rgba(0,0,0,0.3), 0 2px 6px 2px rgba(0,0,0,0.15) | 悬停的卡片，静止的FAB。 |
| 3 | 0 4px 8px 3px rgba(0,0,0,0.15), 0 1px 3px rgba(0,0,0,0.3) | 悬停的FAB，菜单。 |
| 4 | 0 6px 10px 4px rgba(0,0,0,0.15), 0 2px 3px rgba(0,0,0,0.3) | 对话框。 |
| 5 | 0 8px 12px 6px rgba(0,0,0,0.15), 0 4px 4px rgba(0,0,0,0.3) | 导航抽屉。 |

**深色模式**：高度使用色调偏移（更浅的表面色调）代替更重的阴影。这是系统最重要的深色模式原则。

## 宜与忌

宜：
- 从种子源使用动态色彩。这才是它存在的意义。
- 用容器颜色做表面区分，而非阴影。
- 使用20dp按钮圆角——按钮是药丸，不是矩形。
- 使用状态层（透明度覆盖）做交互反馈。
- 让字阶呼吸——展示尺寸用于英雄时刻。

忌：
- 不要用4dp按钮角。那是上一个系统。本系统使用20dp药丸。
- 不要在深色模式中用阴影做高度。用色调偏移代替。
- 不要将上一系统的组件与此系统混用。它们有根本不同的形状和颜色。
- 不要在大面积表面上使用主色。用主色容器代替。
- 除非严格品牌合规，不要用静态hex值覆盖动态色彩。

| 断点 | 宽度 | 导航 | 列数 |
|------|------|------|------|
| 紧凑 | < 600dp | 底部导航栏 | 4 |
