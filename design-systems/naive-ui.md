## 视觉主题与氛围

这个界面说的是"彻底的可定制性，绝不妥协"。哲学很简单：所有的旋钮都在这里——尽管拧。每一个令牌都是响应式的JavaScript值，可以在运行时直接替换，无需切换class。系统不会给你一个"钦定"的主题——它给你的是构建任何主题的机制。

默认亮色主题干净而中性（蓝色#2080F0）。但真正的身份标识是暗色主题：近黑底（#101014）上的薄荷绿（#63E2B7）。这个组合——深邃黑暗中一抹冷调低饱和的薄荷——已经成为开发者工具构建者之间的标志性配色。它在说："这个工具的构建者同时在乎美学和工程。"

系统使用CSS-in-JS，而非CSS变量。这意味着每个令牌都是响应式值——改一处，所有组件即时更新。没有class切换，没有样式表替换，没有FOUC。代价是你不能在自己的CSS里用`var(--token)`——你得用`useThemeVars()`组合式函数。这不是限制，这就是架构。

## 排版规则

字体：拉丁文使用Inter，CJK使用系统回退字体。选择Inter是因为它在小尺寸下渲染出色，以及较宽的x-height，在密集的开发者界面中最大化可读性。

| 级别 | 字号 | 字重 | 为什么 |
|------|------|------|--------|
| H1 | 34px | 600 | "这个页面存在。" 每个视图使用一次。 |
| H2 | 28px | 600 | "这是一个主要分节。" |
| H3 | 24px | 600 | "这是一个话题。" |
| H4 | 20px | 600 | "这是一个分组。" |
| H5 | 16px | 600 | "这是一个强调标签。" |
| H6 | 14px | 600 | "这是一个轻描淡写的标题。" |
| 正文 | 14px | 400 | 默认。14px在1.6行高下——对开发者工具来说足够宽裕。 |
| 小号 | 13px | 400 | 紧凑文字。 |
| 迷你 | 12px | 400 | 下限。 |

字号令牌：fontSizeMini (12px)、fontSizeSmall (13px)、fontSizeMedium (14px)、fontSizeLarge (15px)、fontSizeHuge (16px)。尺寸间1px的增量意味着精准控制——再也不用在"太小"和"太大"之间妥协。

## 组件样式

### 按钮（主色）
- 背景：primaryColor（亮色#2080F0 / 暗色#63E2B7）
- 高度：34px小号，36px默认，38px大号——2px的增量是刻意的，呼应系统精细控制的哲学
- 圆角：3px小号，4px默认，5px大号——每种尺寸有自己的圆角
- 字体：13px小号，14px默认，15px大号，字重500
- 悬停：primaryColorHover
- 焦点：0 0 0 2px rgba(32,128,240,0.3)（亮色）/ rgba(99,226,183,0.3)（暗色）
- 禁用：不透明度0.5

### 卡片
- 背景：var(--color)——自动适配主题
- 边框：无
- 圆角：8px
- 内边距：20px
- 悬停：var(--hoverColor)

### 输入框
- 高度：28px小号，34px默认，40px大号
- 圆角：3/4/5px——与按钮圆角按尺寸一一对应
- 焦点：primaryColor边框+焦点环
- 错误：errorColor边框

## 布局原则

- 基础单位：4px
- 间距：组件各自定义
- 栅格：24列（NGrid），支持响应式span属性：`span="0 s:6 m:8 l:12"`
- 圆角等级：2、3、4、5、6、8、12、50%

| 1 | 卡片 | 微妙阴影 |

## 宜与忌

宜：
- 使用themeOverrides来定制。这才是它存在的意义。
- 为开发者工具启用暗色薄荷主题——这就是身份。
- 使用n-config-provider实现层级化主题。父级覆盖会级联到子级。
- 使用useThemeVars()组合式函数获取响应式令牌。
- 使用tree-shaking（具名导入）保持包体积精简。

忌：
- 不要直接使用CSS自定义属性——本系统使用CSS-in-JS，不是--var()令牌。
- 不要与组件API对抗——通过props和themeOverrides定制，而非CSS覆盖。
- 不要导入全部组件——使用具名导入以支持tree-shaking。
- 不要在同一分节中使用超过2级标题。
- 不要全局覆盖borderRadius——按组件逐个设置才能精细控制。

## 响应式行为

| 断点 | 宽度 | 发生什么 |
|------|------|----------|
| 移动端 | < 640px | 单列，堆叠 |
| 平板 | ≥ 640px | 2列，可折叠侧边栏 |
| 桌面 | ≥ 1024px | 完整布局含侧边栏 |
| 宽屏 | ≥ 1280px | 扩展布局 |

- 触控目标最小：34px
- 字号不低于12px
- NGrid响应式：`span="0 s:6 m:8 l:12"`
