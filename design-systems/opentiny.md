# OpenTiny — 设计系统视觉宪法

## 视觉主题与氛围

**沉默的权威——企业基础设施的暗色底座**

OpenTiny 不用彩色说话，用灰度说话。主色 `#191919`——几乎是黑色——是按钮、激活态、锚点滑块的唯一答案。这不是"没有设计"，这是"设计即克制"的极端宣言。

当所有系统都在用蓝、紫、橙争夺注意力时，OpenTiny 把彩色降到功能角色：`#1476FF` 是链接和信息，`#5CB300` 是成功，`#FF8800` 是警告，`#F23030` 是错误。主按钮是黑底白字，像法官的法槌——不需要颜色来证明权威。

### 核心原则

1. **黑白即品牌**：主操作按钮 `#191919` → `#595959`（hover），从黑到深灰，没有色相偏移
2. **彩色即语义**：蓝=信息、绿=成功、橙=警告、红=错误，绝不跨界
3. **4px原子单位**：间距体系完全基于4px倍数，2/4/8/12/16/24/32，没有例外
4. **6px按钮圆角**：`--tv-border-radius-md: 6px`，介于Element Plus的4px和Ant Design的6px之间
5. **企业级无主见**：设计不表达个性，只表达可靠——华为云、运营商后台的场景

### 深色模式策略

OpenTiny 深色模式遵循"品牌色不变、灰色反转"策略：
- 品牌主色 `#191919` 在深色模式下不适用（本身就是深色），改为 `#1476FF` 蓝色作为操作色
- 灰度体系整体上移：`#F5F5F5` → `#242424`，`#FFFFFF` → `#181818`
- 文字从黑色系切换到白色系：`rgba(0,0,0,0.9)` → `rgba(255,255,255,90%)`

## 排版规则

### 字体栈

```css
--tv-font-family: 'Helvetica', 'Arial', 'PingFangSC-Regular',
                  'Hiragino Sans GB', 'Microsoft YaHei', '微软雅黑',
                  'Microsoft JhengHei';
```

Helvetica 优先——企业级、无主见、跨平台安全。不是Inter，不是Roboto，是老牌瑞士无衬线的沉默。

### 字阶表

| 级别 | 字号 | 字重 | 用途 |
|------|------|------|------|
| heading-xl | 32px | 600 | 页面主标题 |
| heading-lg | 24px | 600 | 区块标题 |
| heading-md | 20px | 600 | 卡片标题 |
| heading-sm | 18px | 600 | 子标题 |
| heading-xs | 16px | 600 | 小标题 |
| default | 14px | 400 | 正文——所有系统默认 |
| sm | 12px | 400 | 辅助文字、标签 |

行高固定 `1.5`，字重只有两档：`400`（常规）和 `600`（强调），没有300，没有700+。

## 布局原则

### 基础单位：4px

```
--tv-space-base: 4px
--tv-space-xs:   2px    (0.5x)
--tv-space-sm:   4px    (1x)
--tv-space-md:   8px    (2x)
--tv-space-lg:   12px   (3x)
--tv-space-xl:   16px   (4x)
--tv-space-xxl:  24px   (6x)
--tv-space-xxxl: 32px   (8x)
```

### 圆角等级

```
--tv-border-radius-xs:    2px   (标签、徽标)
--tv-border-radius-sm:    4px   (小元素)
--tv-border-radius-md:    6px   (按钮、输入框)
--tv-border-radius-lg:    8px   (卡片、Alert)
--tv-border-radius-round: 999px (药丸形)
```

### 高度体系

```
--tv-size-height-xs: 24px  (迷你)
--tv-size-height-sm: 28px  (小)
--tv-size-height-md: 32px  (默认)
--tv-size-height-lg: 40px  (中)
--tv-size-height-xl: 48px  (大)
```

### 阴影层级表

| 级别 | 用途 | 阴影值 | 原因 |
|------|------|--------|------|
| shadow-0 | 通用浮层 | `0 4px 16px 0 rgba(0,0,0,0.08)` | 标准投射——不强调，不消失 |

## 宜与忌

### 必须做 ✓

- 主按钮使用黑底白字 `#191919` / `#FFFFFF`
- 间距使用4px倍数体系
- 圆角使用预定义等级（2/4/6/8/999px）
- 彩色仅用于语义角色（信息蓝/成功绿/警告橙/错误红）
- hover态使用灰度变化（黑→深灰），不用色相变化

### 绝不做 ✗

- 不要用品牌蓝色 `#1476FF` 做主按钮——蓝是信息，不是行动
- 不要随意发明间距值——只有2/4/8/12/16/24/32
- 不要使用超过两种字重——只有400和600
- 不要给阴影加色彩倾向——纯黑透明度就够了
- 不要在深色模式下简单反转——品牌色从黑切蓝，灰度整体上移

---

## 响应式行为

OpenTiny 主要服务于企业后台桌面端，不定义移动端断点。

- 最小内容宽度：1024px
- 侧边栏折叠宽度：64px
- 内容区自适应剩余空间
