## 视觉主题与氛围

这个界面说的是"内容优先的效率"。UI应该消隐于它所承载的内容之后。每个表面、每种颜色、每次交互都为了让用户的工作流更快——而非展示设计系统。

标志性动作是DSM：设计与代码之间的双向桥梁，令牌是活着的契约，而非静态的规格。当设计师在Figma中调整令牌，代码同步更新。当开发者在代码中修改令牌，Figma同步反映。这不是一个功能——这是哲学。设计系统是一场对话，不是独白。

紫色强调（#6B53F7）是刻意的：它说"这是创意工具"而不喊叫。蓝色说企业。紫色说创造。低饱和度的中性色让用户内容呼吸——因为在创意工具中，用户的内容是唯一应该鲜艳的东西。

### 深色模式

虚空是#141414。卡片在#1C1C1E。文字变为#ECECEC。紫色仍是紫色。本系统为深色模式而生——DSM令牌自动处理切换。

## 排版规则

字体：拉丁文使用Inter，CJK使用系统字体。选择Inter是因为它在小尺寸下渲染优美，且x-height最大化密集布局中的可读性。

| 级别 | 字号 | 字重 | 为什么 |
|------|------|------|--------|
| H1 | 32px | 700 | "这是页面的身份。" 700字重因为H1必须锚定——每页只有一个。 |
| H2 | 28px | 700 | "这是一个主要分节。" 仍然粗体因为分节需要地标。 |
| H3 | 24px | 600 | "这是一个话题。" 600字重——不如H2激进，但仍然结构化。 |
| H4 | 20px | 600 | "这是一个分组。" 仍然统帅的最小标题。 |
| H5 | 16px | 600 | "这是一个强调的标签。" 在标题与正文之间的边界上。 |
| H6 | 14px | 600 | "这是一个轻描淡写的标题。" 用于行内分节断点。 |
| 正文 | 14px | 400 | 默认。14px在1.5行高下阅读舒适。 |
| 小字 | 12px | 400 | 元数据、说明文字、辅助文字。 |

标题分6级因为创意工具有深层内容层级。企业工具可以止于3级。

## 组件样式

### 按钮（主色）
- 背景：紫色（#6B53F7）——"创造点什么"
- 高度：32px默认，24px小，40px大
- 圆角：3px默认，6px大——紧凑的角落营造精准、工具般的触感。不是圆角——圆角感觉随意。
- 字体：14px，字重600——比多数系统更重，因为创意界面需要自信的可供性
- 悬停：#7B65F8——"我准备好了"
- 焦点：0 0 0 2px rgba(107,83,247,0.2)——紫色光环
- 禁用：不透明度0.3——重度虚化，因为不可用的创意选项应该强烈退隐

### 卡片
- 背景：白色
- 边框：无——仅通过阴影提供高度。卡片漂浮，它们不是待在盒子里。
- 圆角：8px——成比例的弧度营造容器感
- 阴影：0 4px 14px rgba(0,0,0,0.06)——存在但柔和，因为创意内容需要感觉被抬升而非被围困

### 输入框
- 高度：32px默认
- 圆角：3px——匹配按钮语言。紧凑的角落说"这是工具"。
- 焦点：主色边框，无光环——极简框架因为内容比输入框更重要

### 表格
- 表头：--color-fill-0——最微妙的区分
- 悬停：--color-fill-0——一缕背景变化
- 字体：14px正文，数字用等宽字体——创意工具中的数字是数据，数据值得等宽

## 布局原则

- 基础单位：4px
- 间距比例：4、8、12、16、20、24、32、40、48、64
- 网格：12列——比企业系统列数少，因为创意布局需要更宽的间距和更多留白
- 圆角比例：3px（按钮）→ 6px（大按钮）→ 8px（卡片）→ 12px（弹窗）。增量小因为本系统看重精确胜过温暖。

## 深度与高度

| 级别 | 用途 | 阴影 | 为什么 |
|------|------|------|--------|
| 1 | 卡片 | 0 4px 14px rgba(0,0,0,0.06) | 柔和的抬升。卡片在页面之上但不主导。 |
| 2 | 下拉菜单 | 0 6px 20px rgba(0,0,0,0.08) | 临时元素需要更多存在感。 |
| 3 | 弹窗 | 0 10px 32px rgba(0,0,0,0.12) | 弹窗命令注意力——它是屏幕上最高层的东西。 |

## 宜与忌

宜：
- 保持内容突出。UI支撑内容；永远不与内容竞争。
- 刻意使用紫色（#6B53F7）。它承载创意能量——不要用竞争性强调色稀释它。
- 同时设计浅色和深色。DSM令牌处理切换——设计一次，部署两次。
- 使用原始→语义令牌层进行定制。在主题级别覆盖，永远不要在组件中。

忌：
- 不要在冷调紫色旁边使用暖调强调色。温度一致性很重要——暖色红挨着冷色紫制造视觉不和谐。
- 不要在卡片中添加装饰元素。阴影提供深度；边框会添加噪音。
- 不要在单个组件中覆盖令牌值。DSM是活着的契约——在主题层覆盖，不是在实例层。
- 不要在数据容器上使用圆角（半径>8px）。数据要精确；圆润暗示好玩。
- 每个页面分节不要使用超过2级标题。深层嵌套意味着混乱的层级。

## 响应式行为

| 断点 | 宽度 | 发生什么 |
|------|------|----------|
| 移动端 | < 640px | 单列，堆叠 |
| 平板 | ≥ 640px | 2列，侧边栏覆盖层 |
| 桌面 | ≥ 1024px | 完整布局，常驻导航 |
| 宽屏 | ≥ 1440px | 最大宽度容器 |

- 最小触摸目标：32px
- 字号永远不低于12px

## 代理提示指南

快速调色板：primary=#6B53F7, bg=#FFFFFF, bg1=#F9F9F9, text=#1C1F23, border=#E1E1E3, success=#00B368, warning=#F5BA2D, danger=#F93920

### 即用提示词：
- "创建一个内容管理仪表盘" → 白色背景，#F9F9F9次表面，8px卡片配阴影（0 4px 14px rgba(0,0,0,0.06)），紫色#6B53F7用于CTA，14px正文，3px按钮圆角
- "构建一个创作者工作室界面" → 偏好深色模式（#141414背景），#ECECEC文字，紫色强调，内容优先12列网格，紧凑3px圆角
- "设计一个设置面板" → 分组卡片，使用主色的切换开关，紧凑32px输入框，次文本#878D96
